# 시행착오
import Callout from 'nextra-theme-docs/callout'

## A. 문제
컴포넌트 내에서 `useSelector()`를 **호출하여 받은 값**을 렌더링 하고자 했더니  
<Callout type="error" emoji="🚫">
  Cannot read property '~' of undefined
</Callout>
라는 자주 만나는 에러가 떴다.

## B. 원인
console.log() 를 찍어 살펴보니 useSelector() 앞뒤로 렌더링이 2번 되고 있었다.  

useSelector()는 컴포넌트가 렌더링 된 후에 실행되는데  
아직 `undefined`인 변수를 렌더링 하려고 하는게 원인이었다.

## C. 해결
영어로 short-circuit evaluation(단축 평가)을 논리 연산자를 활용해 해결하는 방법이 있다.  
```jsx
<div>
  {
    value
    && value.map((bla) => <span>{bla}</span>)
  }
</div>
```
이런 방식으로 해결이 가능하다.

아니면 비슷한 방식으로 삼항연산자도 사용해여 해결해도 된다.
