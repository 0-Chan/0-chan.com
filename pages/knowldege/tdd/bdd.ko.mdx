# BDD란?
## 1. BDD 방법론
BDD란 Behavior Driven Development의 약자이며,  
이 개발 방법론은 TDD의 확장이라 볼 수 있습니다.

테스트, 문서, 예시를 한데 모아놓은 개념으로 **시나리오**를 기반으로 테스트 케이스를 작성하는데요.  
쉽게 말하면 테스트 대상을 주인공 삼아 상세히 설명하는 테스트를 작성하는 것입니다.

보통 3개의 핵심 키워드로 패턴이 진행됩니다. 

## 2. Describe - Context - It
Given - When - Then 키워드도 있지만,  
저는 Describe - context - It 키워드를 선호합니다.

각 요소를 설명하면
|키워드|설명|
|---|:----:|
|Describe|대상을 명시|
|Context|대상의 상황 설명|
|It|대상의 행동 설명|

- describe:  
테스트 대상 이름을 명시합니다.    
클래스, 함수, 컴포넌트 등을 짧고 분명하게 작성합니다.  
또 Describe는 하위 블록을 모으는 역할을 합니다.
- context:  
테스트를 구분하기 쉽게 조건을 나누는 단계입니다.  
보통 상반되는 상황이나 입력 파라미터를 설명합니다.  
`when` 또는 `with` 또는 `without` 으로 시작합니다. 예를 들면
```js
context('when logged in', () => {
  it('blabla...
});

context('when logged out', () => {
  it('blabla...
});
```
그리고 만약 상반된 상황이 없다면 context는 생략하고 바로 it문으로만 작성합니다.
- it:  
대상이 무엇을 리턴하는지, 수행하는 목표를 설명합니다.  
it 블록 안에는 expect가 딱 하나 들어가게 됩니다.

## 3. 장점
이 방식으로 테스트 코드를 작성하면 2가지 이점이 있습니다.  
- 테스트 대상, 조건, 결과를 확실하게 구분해서 볼 수 있다!
- 읽기가 수월해서 수정 및 디버깅 하기가 비교적 쉬워진다!  

이러한 장점 덕분에 시간이 지나 요구 사항이 바뀌어도 이에 대응하기가 훨씬 간편합니다.
- - -
p.s.) Jest에서 context문을 쓰려면 `jest-plugin-context`을 설치하고 `.eslintrc.js`에 설정을 해줘야합니다.
