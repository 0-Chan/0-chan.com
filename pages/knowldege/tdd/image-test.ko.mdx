# 이미지를 테스트
## Ⅰ. 테스트 대상 2가지
이미지를 테스트한다고 할 때, 2가지 대상이 있습니다.  
ⅰ. 이미지가 존재하는지?  
ⅱ. attribute 값이 올바른지?

### ⅰ. 이미지가 존재하는지?
만약 `import`로 이미지를 불러오는 경우가 있습니다.  
이 때 파일의 없거나 정확한 경로 값이 아니라면 빌드 단에서 에러가 납니다.  

이 외에는 [toBeVisible()](https://github.com/testing-library/jest-dom#tobevisible)를 써서 실제 유저에게 보여지는지 테스트합니다.  
img element가 `hidden` 속성이나 css 등으로 숨겨질 경우가 있기 때문입니다.  
ex)
```js
describe('Img', () => {
  it('img is visible', () => {
    const { getByRole } = render(<Image />);

    expect(getByRole('img')).toBeVisible();
  });
});
```

### ⅱ. attribute 값이 올바른지?
img 속성 값인 `src`나`alt` 값이 올바른지 테스트합니다.  
ex)
```js
describe('Img', () => {
  it('img contains correct attribute values', () => {
    const { getByRole } = render(<Image />);

    expect(getByRole('img')).toHaveAttribute('src', 'images/blabla.png');
    expect(getByRole('img')).toHaveAttribute('alt', 'blabla');
  });
});
```
이렇게 2개 방법으로 이미지를 테스트 할 수 있습니다.
